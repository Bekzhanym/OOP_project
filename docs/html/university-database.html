kbtu_oop_project.infrastructure.persistence.UniversityDatabase

Роль: единая точка хранения состояния приложения и операций над ним.

  • Пользователи и курсы; восстановление связей «преподаёт» и «записан» после загрузки.
  • Аудит: recordAudit — текстовые строки с датой; recordStructured(Log) — записи с действием и userId.
  • Регистрация на курс: очередь PendingCourseRegistration, утверждение/отклонение менеджером.
  • Почта сотрудников: EmployeeMessage, фильтры входящих и флаг «декан».
  • Проекты: глобальный список ResearchProject для каталога и присоединения студентов.
  • Сериализация: пользователи, курсы, строковый аудит, список Log, pending-заявки, проекты, почта сотрудников, новости; файл data/university-state.ser; хвост через readOptionalSnapshotTail.
