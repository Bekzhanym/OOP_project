Сводка «тройки»: ТЗ, код и диаграмма

Код приведён к общим требованиям и к элементам вашего partA (3).mdj. Файл .mdj лучше править в StarUML (Reverse Engineering / Model Explorer), а не вручную в текстовом редакторе.

1. Что уже совпадает в коде с ТЗ и UML

  • CourseType связан с Course (поле + консоль менеджера/преподавателя + демо).
  • Lesson + LessonType (Lecture / Practice) задаются при создании курса менеджером и в гостевом демо.
  • Student4thYear: регистрация ролью Role.STUDENT_4TH_YEAR; назначение руководителя через админа (п. 9); исключение при низком h-index.
  • Student.viewTranscript() реализован; пункт студента вызывает его из консоли.
  • Teacher.manageCourse, exportGradeReport, writeRecommendation — рабочие реализации + пункты 11–12 в меню преподавателя.
  • Новости: хранятся в UniversityDatabase, сериализуются, менеджер добавляет; гость и студент читают.
  • Рекомендации: Student.requestRecommendation(Teacher) доступен из меню студента.
  • Research staff: при регистрации запрашивается h-index.

2. Что обязательно отразить в StarUML (reverse engineering)

  1. User — сделать abstract class (не interface), если диаграмма ещё показывает интерфейс.
  2. Явная цепочка: Teacher → Employee → User (и аналогично Admin/Manager).
  3. Добавить классы из реализации демо: PendingCourseRegistration, EmployeeMessage, поля и связи с UniversityDatabase.
  4. UniversityDatabase: актуализировать атрибуты — списки pending-заявок, проектов, почты, структурированных Log, строкового аудита и newsItems.
  5. В Course добавить атрибут courseType: CourseType.
  6. В перечислении ролей / фабрике указать роль для четвёртого курса (STUDENT_4TH_YEAR) или эквивалент на диаграмме.
  7. Удалить с диаграммы заглушки Class1, Interface1, дубликаты Professor, если они не используются.
  8. Сохранить экспорт диаграммы в отчёт (PDF) как требует курс.

3. Быстрая проверка перед защитой

  • Гость → демо → курс с CourseType и LessonType.
  • Регистрация студента 4 курса → админ назначает супервизора с h-index ≥ 3 → ошибка при h-index < 3.
  • Менеджер → добавить курс → новости → утвердить заявку на запись.
  • Преподаватель → manageCourse → exportGradeReport → поставить оценку.
  • Сохранить data/university-state.ser, перезапуск — новости и pending при необходимости на месте.
